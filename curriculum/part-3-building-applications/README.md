# Part III: Building Complete Applications

Progressive development of the Lemonade Stand — from static HTML to deployed fullstack app.

**Full curriculum**: [episk-pos.github.io/DevFoundry/docs/curriculum/part-3-building-applications](https://episk-pos.github.io/DevFoundry/docs/curriculum/part-3-building-applications)

---

## Stages ✓

| Stage | Module | What You Build |
|-------|--------|----------------|
| 1 | [Static Website](./11-static-website.md) | HTML/CSS/JS files, DOM manipulation |
| 2 | [Interactive UI](./12-interactive-ui.md) | Event handlers, state, dynamic updates |
| 3 | [React SPA](./13-react-spa.md) | Components, routing, modern tooling |
| 4 | [Fullstack App](./14-fullstack-app.md) | Express API, database, client-server |
| 5 | [Deployed App](./15-deployed-app.md) | Production hosting, CI/CD |

---

## The Journey

Each stage builds on the previous:

```
Stage 1        Stage 2        Stage 3        Stage 4        Stage 5
Static    →   Interactive  →   React    →   Fullstack  →   Deployed
HTML/CSS/JS    + State        + Components   + API/DB       + Production
```

---

## What You'll Learn

- How to evolve an application through architectural stages
- When each level of complexity is warranted
- All three architectural views (Module, Runtime, Deployment)
- How to use AI assistants effectively at each stage
